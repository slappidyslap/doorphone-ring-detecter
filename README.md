# Doorphone Ring Detector

Проект состоит из трех основных частей:
1.  Микроконтроллер ESP32, который отслеживает звук звонка домофона.
2.  Java сервер (модуль `core`), который принимает события от ESP32.
3.  Java приложение с GUI в системном трее (модуль `tray`), которое запускает сервер и позволяет взаимодействовать с ним.

## Протокол обмена данными

Обмен данными между сервером и IoT-устройством осуществляется по простому бинарному протоколу через TCP.

Структура каждого сообщения (команды или события):
`[1 байт - тип] [4 байта - длина данных] [N байт - данные]`

*   **Тип:** Один байт, идентифицирующий тип сообщения.
    *   `0x00` - `0x7F`: Команды (Сервер -> Устройство)
    *   `0x80` - `0xFF`: События (Устройство -> Сервер)
*   **Длина данных:** 4-байтовое целое число (big-endian), указывающее длину полезной нагрузки в байтах.
*   **Данные:** Непосредственно полезная нагрузка сообщения.

### Типы сообщений

#### Команды (Сервер -> Устройство)

*   **`0x01`: `PingCommand`**
    *   **Данные:** Пустые (0 байт).
    *   Используется для проверки, активно ли устройство. Ожидается, что устройство ответит немедленно.

#### События (Устройство -> Сервер)

*   `0x81`: `DoorphoneRingDetectedEvent`
    *   **Данные:** 8-байтовое целое число `long` (big-endian), представляющее временную метку UTC (в миллисекундах), когда был обнаружен звонок.
